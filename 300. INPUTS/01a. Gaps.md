# Odwoływanie się do kolumn

Jak prawidłowo odwołać się do kolumny Kwota w tabeli Sprzedaz w języku DAX?

## Dostępne funkcje
- Sprzedaz.Kwota
- [Kwota]
- Sprzedaz[Kwota]
- Kwota

## Kod do uzupełnienia

```dax
DAX
Suma Sprzedaży = SUM([SLOT:0])
```

## Poprawne rozwiązanie
Sprzedaz[Kwota]

## Feedback poprawny
✅ Brawo! Rozwiązanie poprawne!

W DAX zawsze odwołujesz się do kolumn poprzez nazwę tabeli i nazwę kolumny w nawiasach kwadratowych: Tabela[Kolumna]. To zapobiega niejednoznaczności gdy masz kolumny o tych samych nazwach w różnych tabelach.

## Wskazówka
W DAX używamy nawiasów kwadratowych, a nie kropki. Format to: NazwaTabeli[NazwaKolumny].

# Nazwy z apostrofami

Masz tabelę o nazwie Tabela Sprzedaży (ze spacją) i kolumnę Kwota Netto. Jak prawidłowo się do niej odwołać?

## Dostępne funkcje
- Tabela Sprzedaży[Kwota Netto]
- ´Tabela Sprzedaży´[Kwota Netto]
- [Kwota Netto]
- [Tabela Sprzedaży][Kwota Netto]

## Kod do uzupełnienia

```dax
DAX
Suma = SUM([SLOT:0])
```

## Poprawne rozwiązanie
Tabela z apostrofem[Kwota Netto]

## Feedback poprawny
✅ Brawo! Rozwiązanie poprawne!

Gdy nazwa tabeli lub kolumny zawiera spacje, znaki specjalne lub polskie znaki, musisz ująć ją w apostrofy. Poprawna składnia to: nazwa z apostrofem. Nazwa kolumny w tym przypadku nie wymaga apostrofów, bo jest już w nawiasach kwadratowych.


## Wskazówka
Gdy nazwa zawiera spacje, musisz ją ująć w specjalne znaki.

# Funkcja agregująca SUM

Chcesz obliczyć sumę wszystkich wartości w kolumnie Kwota z tabeli Sprzedaz. Która funkcja jest odpowiednia?

## Dostępne funkcje
- AVERAGE
- SUM
- COUNT
- SUMX

## Kod do uzupełnienia

```dax
DAX
Suma Sprzedaży = [SLOT:0](Sprzedaz[Kwota])
```

## Poprawne rozwiązanie
SUM

## Feedback poprawny
✅ Brawo! Rozwiązanie poprawne!

Funkcja SUM() sumuje wszystkie wartości w kolumnie. Pomija wartości puste (BLANK) i uwzględnia aktualny kontekst filtrowania. To podstawowa funkcja agregująca w DAX.

## Wskazówka
Szukasz podstawowej funkcji agregującej, której nazwa w języku angielskim oznacza suma.

# Funkcja IF

Chcesz utworzyć kategorię sprzedaży: jeśli kwota > 1000, to Wysoka, w przeciwnym razie Niska. Jak uzupełnić formułę?

## Dostępne funkcje
- IF
- CASE
- ″Wysoka″
- ″Niska″

## Kod do uzupełnienia

```dax
DAX
Kategoria =
[SLOT:0](
    Sprzedaz[Kwota] > 1000,
    [SLOT:1],
    [SLOT:2]
)
```

## Poprawne rozwiązanie
IF, Wysoka, Niska

## Feedback poprawny
✅ Brawo! Rozwiązanie poprawne!

Funkcja IF() w DAX ma składnię: `IF(warunek, wartość_jeśli_prawda, wartość_jeśli_fałsz).` W tym przypadku: `IF(Sprzedaz[Kwota] > 1000, ″Wysoka″, ″Niska″).` 

## Wskazówka
W DAX funkcja warunkowa to po prostu IF z trzema parametrami: warunek, co zwrócić gdy prawda, co zwrócić gdy fałsz.

# Funkcja SUMX

Chcesz obliczyć sumę wartości zamówień (Ilość × Cena) dla każdego wiersza. Którą funkcję powinieneś użyć?

## Dostępne funkcje
- SUM
- SUMX
- CALCULATE
- TOTALX

## Kod do uzupełnienia

```dax
DAX
Suma Wartości =
[SLOT:0](
    Sprzedaz,
    Sprzedaz[Ilość] * Sprzedaz[Cena]
)
```

## Poprawne rozwiązanie
SUMX

## Feedback poprawny
✅ Brawo! Rozwiązanie poprawne!

SUMX() to funkcja iteracyjna, która dla każdego wiersza tabeli oblicza wyrażenie, a potem sumuje wyniki. Składnia: SUMX(tabela, wyrażenie). W tym przypadku dla każdego wiersza mnoży Ilość × Cena, a następnie sumuje wszystkie wyniki.
## Wskazówka
Potrzebujesz funkcji iteracyjnej (z końcówką X), która sumuje wyniki obliczeń dla każdego wiersza.

# Operator łączenia tekstów

Chcesz połączyć imię i nazwisko klienta ze spacją pomiędzy. Jaki operator użyć?

## Dostępne funkcje
- +
- &
- !
- CONCAT

## Kod do uzupełnienia

```dax
DAX
Pełna Nazwa =
Klienci[Imię] [SLOT:0] Klienci[Nazwisko]
```

## Poprawne rozwiązanie
&

## Feedback poprawny
✅ Brawo! Rozwiązanie poprawne!

Operator ampersand w DAX służy do łączenia (konkatenacji) tekstów. Poprawna składnia: `Klienci[Imię] &  Klienci[Nazwisko].` Możesz łączyć dowolną liczbę elementów tekstowych używając tego operatora.
