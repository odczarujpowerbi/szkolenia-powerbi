# Separator argumentów w DAX

W angielskiej wersji językowej Power BI chcesz napisać funkcję SUMX. Jaki separator powinieneś użyć między argumentami funkcji?

## Dostępne funkcje
- Przecinek
- Średnik
- Dwukropek
- Kropka

## Kod do uzupełnienia

```dax
DAX
Suma wartości = SUMX(fDemo[SLOT:0] fDemo[Price] * fDemo[Quantity])
```

## Poprawne rozwiązanie
Przecinek

## Feedback poprawny
✅ **Brawo! Rozwiązanie poprawne!**

W polskiej wersji Power BI separator argumentów to średnik, a separator dziesiętny to przecinek. Przykład: SUMX(fDemo, fDemo[Price] * fDemo[Quantity]). W angielskiej wersji jest odwrotnie - separator argumentów to przecinek, a dziesiętny to kropka.

## Wskazówka
W polskiej wersji językowej przecinek pełni inną funkcję niż separator argumentów. Pomyśl, jaki znak w języku polskim często oddziela elementy w listach.

# Wyłączenie automatycznego wykrywania typów

Chcesz mieć pełną kontrolę nad typami danych w Power Query i wyłączyć automatyczne wykrywanie. Gdzie znajdziesz tę opcję?

## Dostępne funkcje
- GLOBALNE 
- BIEŻĄCY PLIK
- Power Query
- Ładowanie danych

## Kod do uzupełnienia

```
Plik → Opcje i ustawienia → Opcje → [SLOT:0] → [SLOT:1]
Odznacz: Automatycznie wykrywaj typy kolumn...
```

## Poprawne rozwiązanie
BIEŻĄCY PLIK, Ładowanie danych

## Feedback poprawny
✅ **Brawo! Rozwiązanie poprawne!**

Opcja automatycznego wykrywania typów znajduje się w: Plik → Opcje i ustawienia → Opcje → BIEŻĄCY PLIK → Ładowanie danych. Należy tam odznaczyć opcję Automatycznie wykrywaj typy kolumn i nagłówki dla źródeł nieustrukturyzowanych. To ustawienie dotyczy konkretnego pliku .pbix, nie globalnie całego Power BI.

## Wskazówka
To ustawienie dotyczy procesu ładowania danych do modelu w konkretnym pliku, nie jest to ustawienie globalne programu.

# Widok DAX Query

W którym widoku Power BI możesz pisać i testować zapytania DAX bez tworzenia miar w modelu?

## Dostępne funkcje
- Widok Raport
- Widok Model
- Widok DAX Query
- Widok Dane

## Kod do uzupełnienia

```dax
DAX
-- Ten kod możesz wykonać w: [SLOT:0]
EVALUATE
SUMMARIZECOLUMNS(
    fDemo[Category],
    Total Sales, SUMX(fDemo, fDemo[Price] * fDemo[Quantity])
)
```

## Poprawne rozwiązanie
Widok DAX Query

## Feedback poprawny
✅ **Brawo! Rozwiązanie poprawne!**

Widok DAX Query pozwala na pisanie i testowanie zapytań DAX bezpośrednio, bez konieczności tworzenia miar w modelu. Możesz tam używać słowa kluczowego EVALUATE do eksploracji danych, testowania logiki i nauki składni DAX. Jest to doskonałe narzędzie do debugowania i prototypowania.

## Wskazówka
Szukasz widoku, którego nazwa bezpośrednio odnosi się do języka DAX i możliwości wykonywania zapytań (ang. query).
